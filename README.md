## 1. Подготовка окружения

### 1.1 Установка Python

Убедитесь, что у вас установлен Python версии 3.8 или выше. Проверить версию можно командой:

```bash
python --version
```

### 1.2 Создание виртуального окружения

Виртуальное окружение изолирует зависимости проекта от системных пакетов.

```bash
# Создаём виртуальное окружение с именем venv
python -m venv venv
```

**Что происходит:**
- Создаётся папка `venv` с копиями исполняемых файлов Python
- Устанавливается pip внутри виртуального окружения
- Все пакеты будут устанавливаться только в это окружение

### 1.3 Активация виртуального окружения

**Для Windows:**
```bash
venv\Scripts\activate
```

**Для Linux/macOS:**
```bash
source venv/bin/activate
```

После активации вы увидите `(venv)` в начале строки терминала.

### 1.4 Установка Django

```bash
pip install django
```

**Что происходит:**
- Загружается последняя стабильная версия Django
- Устанавливается вместе со всеми зависимостями
- Команда `django-admin` становится доступной

## 2. Создание проекта Django

### 2.1 Инициализация проекта

```bash
django-admin startproject shoestore .
```

**Важно:** Точка в конце команды означает, что проект создаётся в текущей директории.

**Что происходит:**
- Создаётся структура папок проекта
- Генерируются базовые конфигурационные файлы

### 2.2 Структура проекта после создания

```
shoestore/           # Папка проекта (конфигурация)
    __init__.py      # Пустой файл, делает папку Python-пакетом
    asgi.py          # Конфигурация ASGI для асинхронных серверов
    settings.py      # Основные настройки проекта
    urls.py          # Корневая маршрутизация URL
    wsgi.py          # Конфигурация WSGI для веб-серверов
manage.py            # Утилита командной строки для управления проектом
```

### 2.3 Файл manage.py

**Путь:** `manage.py`

```python
#!/usr/bin/env python
"""Django's command-line utility for administrative tasks."""
import os
import sys


def main():
    """Run administrative tasks."""
    # Устанавливаем переменную окружения DJANGO_SETTINGS_MODULE
    # Она указывает Django, какой файл настроек использовать
    os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'shoestore.settings')
    try:
        # Импортируем функцию для выполнения команд из командной строки
        from django.core.management import execute_from_command_line
    except ImportError as exc:
        # Обрабатываем ошибку импорта Django
        raise ImportError(
            "Couldn't import Django. Are you sure it's installed and "
            "available on your PYTHONPATH environment variable? Did you "
            "forget to activate a virtual environment?"
        ) from exc
    # Выполняем команду, переданную через аргументы командной строки
    execute_from_command_line(sys.argv)


if __name__ == '__main__':
    main()
```

**Комментарии к коду:**
- `os.environ.setdefault()` — устанавливает переменную окружения, если она ещё не установлена
- `execute_from_command_line()` — основная функция управления Django (запуск сервера, миграции и т.д.)
- `sys.argv` — список аргументов командной строки (например, `['manage.py', 'runserver']`)

## 5. Модели данных

### 5.1 Файл models.py

**Путь:** `products/models.py`

Модели определяют структуру базы данных. Каждая модель — это таблица в БД.

```python
from django.db import models
from django.contrib.auth.models import AbstractUser
# Create your models here.

# Модель категории товаров
class Category(models.Model):
    # CharField — текстовое поле фиксированной длины
    # max_length=100 — максимальная длина строки
    # verbose_name='Название' — человеко-читаемое имя поля
    name = models.CharField(max_length=100, verbose_name='Название')

    class Meta:
        # db_table — имя таблицы в базе данных
        db_table = 'category'
        # managed=True — Django управляет таблицей (создаёт, изменяет)
        managed = True
        # verbose_name — человеко-читаемое имя модели в единственном числе
        verbose_name = 'Категория'
        # verbose_name_plural — во множественном числе
        verbose_name_plural = 'Категории'

    def __str__(self):
        # Метод возвращает строковое представление объекта
        # Используется в админке и при печати объектов
        return self.name


# Модель поставщика
class Supplier(models.Model):
    name = models.CharField(max_length=100, verbose_name='Название')

    class Meta:
        db_table = 'supplier'
        managed = True
        verbose_name = 'Поставщик'
        verbose_name_plural = 'Поставщики'

    def __str__(self):
        return self.name


# Модель производителя
class Manufacturer(models.Model):
    name = models.CharField(max_length=100, verbose_name='Название')

    class Meta:
        db_table = 'manufacturer'
        managed = True
        verbose_name = 'Производитель'
        verbose_name_plural = 'Производитеи'

    def __str__(self):
        return self.name


# Модель товара
class Product(models.Model):
    # Артикул — уникальный идентификатор товара
    article = models.CharField(max_length=50, unique=True, verbose_name='Артикул')
    
    # Наименование товара
    name = models.CharField(max_length=100, verbose_name='Наименование')
    
    # Единица измерения (шт., кг, м и т.д.)
    # default='шт.' — значение по умолчанию
    unit_type = models.CharField(max_length=50, default='шт.', verbose_name='Единица измерения')
    
    # Цена товара
    # max_digits=10 — всего цифр, decimal_places=2 — знаков после запятой
    price = models.DecimalField(max_digits=10, decimal_places=2, verbose_name='Цена')
    
    # ForeignKey — связь «многие к одному»
    # Many products can have one supplier
    # on_delete=models.PROTECT — запрещает удаление поставщика, если есть товары
    supplier = models.ForeignKey(Supplier, verbose_name='Поставщик', on_delete=models.PROTECT)
    
    # Производитель товара
    manufacturer = models.ForeignKey(Manufacturer, verbose_name='Производитель', on_delete=models.PROTECT)
    
    # Категория товара
    category = models.ForeignKey(Category, verbose_name='Категория товара', on_delete=models.PROTECT)
    
    # Скидка в процентах
    discount = models.DecimalField(max_digits=5, decimal_places=2, default=0, verbose_name='Действующая скидка')
    
    # Количество на складе
    stock_quantity = models.IntegerField(verbose_name='Кол-во на складе')
    
    # Описание товара
    # TextField — поле для большого текста
    # max_length=1024 — ограничение длины (не всегда соблюдается в SQLite)
    description = models.TextField(max_length=1024, verbose_name='Описание товара')
    
    # Путь к фотографии
    # blank=True — поле необязательно при сохранении
    # null=True — может хранить NULL в БД
    photo = models.CharField(max_length=255, blank=True, null=True, verbose_name='Путь к фотографии')

    class Meta:
        db_table = 'product'
        managed = True
        verbose_name = 'Товар'
        verbose_name_plural = 'Товары'

    def __str__(self):
        # Возвращает строку вида "Артикул Название"
        return f"{self.article} {self.name}"

    @property
    def price_after_discount(self):
        # Вычисляемое свойство: цена со скидкой
        # Формула: цена * (1 - скидка / 100)
        return self.price * (1 - self.discount / 100)


# Кастомная модель пользователя
# Наследуется от AbstractUser для расширения стандартной модели
class User(AbstractUser):
    # Выбор ролей пользователя
    ROLE_CHOICES = [
        ('client', 'Авторизованный клиент'),
        ('manager', 'Менеджер'),
        ('admin', 'Администратор'),
    ]
    
    # Поле роли с выбором из списка
    role = models.CharField(max_length=50, choices=ROLE_CHOICES, verbose_name='Роль')

    @property
    def is_admin_user(self):
        # Проверка: является ли пользователь администратором
        return self.role == 'Администратор'

    @property
    def is_manager(self):
        # Проверка: является ли пользователь менеджером
        return self.role == 'Менеджер'

    @property
    def is_team(self):
        # Проверка: является ли пользователь частью команды (менеджер или админ)
        return self.role in ['Менеджер', 'Администратор']
```

**Комментарии к типам полей:**
- `CharField` — строка фиксированной длины
- `IntegerField` — целое число
- `DecimalField` — число с плавающей точкой (для денег)
- `TextField` — большое текстовое поле
- `ForeignKey` — связь с другой таблицей
- `BooleanField` — true/false

**Комментарии к параметрам полей:**
- `max_length` — максимальная длина строки
- `unique=True` — значения должны быть уникальны
- `blank=True` — поле необязательно в формах
- `null=True` — поле может быть NULL в БД
- `default=` — значение по умолчанию
- `verbose_name` — читаемое имя поля
- `on_delete` — что делать при удалении связанного объекта

## 6. Миграции

### 6.1 Создание миграций

Миграции — это способ синхронизировать модели с базой данных.

```bash
python manage.py makemigrations
```

**Что происходит:**
- Django анализирует изменения в моделях
- Создаётся файл миграции в `products/migrations/`
- Файл содержит инструкции по изменению БД

### 6.2 Применение миграций

```bash
python manage.py migrate
```

**Что происходит:**
- Применяются все неприменённые миграции
- Создаются таблицы в базе данных
- Записывается информация о применённых миграциях

```python
from django import forms
from .models import Product

# Форма для создания/редактирования товара
# ModelForm автоматически создаёт форму на основе модели
class ProductForm(forms.ModelForm):

    class Meta:
        # Указываем модель, на основе которой создаётся форма
        model = Product
        
        # Поля, которые будут включены в форму
        fields = (
            'article',        # Артикул
            'name',           # Наименование
            'unit_type',      # Единица измерения
            'price',          # Цена
            'supplier',       # Поставщик (выпадающий список)
            'manufacturer',   # Производитель (выпадающий список)
            'category',       # Категория (выпадающий список)
            'discount',       # Скидка
            'stock_quantity', # Количество на складе
            'description',    # Описание
            'photo'           # Фотография
        )

        # Виджеты — HTML-элементы для полей формы
        widgets = {
            # TextInput — обычное текстовое поле
            'article': forms.TextInput(),
            'name': forms.TextInput(),
            'unit_type': forms.TextInput(),
            
            # NumberInput — поле для чисел
            # step='0.01' — шаг изменения (для десятичных дробей)
            'price': forms.NumberInput(attrs={'step': '0.01'}),
            
            # Select — выпадающий список
            'supplier': forms.Select(),
            'manufacturer': forms.Select(),
            'category': forms.Select(),
            
            # Поле скидки с шагом 0.01
            'discount': forms.NumberInput(attrs={'step': '0.01'}),
            
            # Количество — целое число
            'stock_quantity': forms.NumberInput(),
            
            # Textarea — многострочное текстовое поле
            # rows=3 — высота в строках
            'description': forms.Textarea(attrs={'rows': 3}),
            
            # Поле для пути к фото
            'photo': forms.TextInput()
        }
```

**Комментарии:**
- `ModelForm` — класс для создания форм на основе моделей
- `fields` — список полей модели, включаемых в форму
- `widgets` — настройка HTML-представления полей
- `attrs` — HTML-атрибуты элемента (step, rows и т.д.)

## 8. Представления (Views)

### 8.1 Файл views.py

**Путь:** `products/views.py`

Представления обрабатывают HTTP-запросы и возвращают ответы.

```python
from django.shortcuts import render, redirect, get_object_or_404
from .models import Product, Supplier
from .forms import ProductForm


# Представление списка товаров
def product_list(request):
    """
    Отображает список всех товаров.
    Доступно всем пользователям.
    """
    # Получаем все товары из базы данных
    products = Product.objects.all()
    
    # Получаем всех поставщиков (для фильтра)
    suppliers = Supplier.objects.all()

    # Рендерим шаблон с данными
    # request — объект запроса
    # 'products/product_list.html' — путь к шаблону
    # {dict} — контекст (переменные для шаблона)
    return render(request, 'products/product_list.html', {'products': products, 'suppliers': suppliers})


# Представление добавления товара
def product_add(request):
    """
    Добавляет новый товар.
    Доступно только администраторам.
    """
    # Проверка прав: только администраторы могут добавлять товары
    if not request.user.is_admin_user:
        # Перенаправляем на список товаров если нет прав
        return redirect('products:product_list')

    # Если запрос POST (форма отправлена)
    if request.method == 'POST':
        # Создаём форму с данными из запроса
        form = ProductForm(request.POST)
        
        # Проверяем валидность формы
        if form.is_valid():
            # Сохраняем товар в базу данных
            form.save()
            # Перенаправляем на список товаров
            return redirect('products:product_list')
    else:
        # Если GET-запрос — создаём пустую форму
        form = ProductForm()
    
    # Рендерим форму
    return render(request, 'products/product_form.html', {'form': form, 'title': 'Добавление товара'})


# Представление редактирования товара
def product_edit(request, pk):
    """
    Редактирует существующий товар.
    pk — первичный ключ товара (из URL)
    Доступно только администраторам.
    """
    # Проверка прав
    if not request.user.is_admin_user:
        return redirect('products:product_list')

    # Получаем товар или возвращаем 404 если не найден
    product = get_object_or_404(Product, pk=pk)

    # Если POST-запрос
    if request.method == 'POST':
        # Создаём форму с данными и привязываем к существующему товару
        form = ProductForm(request.POST, instance=product)
        
        # Проверяем и сохраняем
        if form.is_valid():
            form.save()
            return redirect('products:product_list')
    else:
        # GET-запрос — заполняем форму текущими данными товара
        form = ProductForm(instance=product)
    
    return render(request, 'products/product_form.html', {'form': form, 'title': 'Редактирование товара'})


# Представление удаления товара
def product_delete(request, pk):
    """
    Удаляет товар.
    pk — первичный ключ товара
    Доступно только администраторам.
    """
    # Проверка прав
    if not request.user.is_admin_user:
        return redirect('products:product_list')

    # Получаем товар или 404
    product = get_object_or_404(Product, pk=pk)

    # Если POST-запрос (подтверждение удаления)
    if request.method == 'POST':
        # Удаляем товар
        product.delete()
        # Перенаправляем на список
        return redirect('products:product_list')

    # GET-запрос — показываем страницу подтверждения
    return render(request, 'products/product_confirm_delete.html', {'product': product})
```

**Комментарии:**
- `render()` — рендерит HTML-шаблон с контекстом
- `redirect()` — перенаправляет на другой URL
- `get_object_or_404()` — получает объект или возвращает 404
- `request.user` — текущий пользователь
- `request.method` — тип HTTP-запроса (GET/POST)
- `pk` — первичный ключ (primary key) из URL

```python
"""
URL configuration for products application.

The `urlpatterns` list routes URLs to views.
"""
from django.urls import path
from . import views

# Пространство имён для URL этого приложения
# Позволяет различать URL разных приложений
app_name = 'products'

urlpatterns = [
    # Главная страница — список товаров
    # '' — пустой путь (корень приложения)
    # views.product_list — функция представления
    # name="product_list" — имя URL для использования в шаблонах
    path('', views.product_list, name="product_list"),
    
    # Страница добавления товара
    # 'product/add/' — URL路径
    path('product/add/', views.product_add, name="product_add"),
    
    # Страница удаления товара
    # '<int:pk>' — захватывает числовой параметр pk из URL
    # Например: /product/5/delete/ где pk=5
    path('product/<int:pk>/delete/', views.product_delete, name="product_delete"),
    
    # Страница редактирования товара
    path('product/<int:pk>/edit/', views.product_edit, name="product_edit"),
]
```

**Комментарии:**
- `path()` — функция для определения маршрута
- `app_name` — пространство имён (используется как `products:name_url`)
- `<int:pk>` — конвертер пути (захватывает целое число)

```html
            <td>
                <!-- Ссылка на редактирование -->
                <a href="{% url 'products:product_edit' product.pk%}">Изменить</a>
                <!-- Ссылка на удаление -->
                <a href="{% url 'products:product_delete' product.pk %}">удалить</a>
            </td>
            {% endif %}
        </tr>
        {% endfor %}
    </tbody>
</table>

<!-- JavaScript для фильтрации и сортировки на клиенте -->
<script>
    // Получаем все строки товаров
    let productRows = Array.from(document.querySelectorAll('.product-row'))
    
    // Функция обновления отображения
    function update() {
        // Получаем поисковый запрос
        let query = document.getElementById('search').value.toLowerCase()
        // Получаем выбранного поставщика
        let sup = document.getElementById('supplier').value

        // Фильтруем строки
        let filtered = productRows.filter(row => {
            // Проверка соответствия поисковому запросу
            let metchQuery = row.textContent.toLowerCase().includes(query)
            // Проверка соответствия поставщику
            let matchSup = sup !== 'Все поставщики' ? row.dataset.supplier == sup : true
            return metchQuery && matchSup
        })
        
        // Получаем тип сортировки
        let sort = document.getElementById('sort').value

        // Сортировка по возрастанию
        if (sort == 'asc') filtered.sort((a, b) => a.dataset.stock - b.dataset.stock)
        // Сортировка по убыванию
        if (sort == 'desc') filtered.sort((a, b) => b.dataset.stock - a.dataset.stock)

        // Очищаем таблицу
        let table = document.querySelector('table tbody')
        table.innerHTML = ''

        // Добавляем отфильтрованные строки
        filtered.forEach(row => {
            table.appendChild(row)
        });
    }
    
    // Назначаем обработчики событий
    document.getElementById('search').oninput = update
    document.getElementById('supplier').onchange = update
    document.getElementById('sort').onchange = update
</script>
{% endblock%}
```

### 13.2 Скрипт импорта продуктов

**Путь:** `import_products.py`

```python
import django, csv, os

# Устанавливаем переменную окружения для Django
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'shoestore.settings')

# Инициализируем Django
django.setup()

# Импортируем модели после инициализации Django
from products.models import Product, Supplier, Manufacturer, Category

# Открываем CSV-файл с товарами
# encoding='utf-8-sig' — обработка BOM для корректного чтения кириллицы
with open('data/products.csv', 'r', encoding='utf-8-sig') as f:
    # DictReader читает CSV как словарь (ключи — заголовки столбцов)
    # delimiter=';' — разделитель полей (точка с запятой)
    for row in csv.DictReader(f, delimiter=';'):
        # Получаем или создаём категорию
        # get_or_create возвращает кортеж (объект, создан_ли)
        cat, _ = Category.objects.get_or_create(name=row['Категория товара'].strip())
        
        # Получаем или создаём поставщика
        sup, _ = Supplier.objects.get_or_create(name=row['Поставщик'].strip())
        
        # Получаем или создаём производителя
        man, _ = Manufacturer.objects.get_or_create(name=row['Производитель'].strip())

        # Получаем или создаём товар
        # article — уникальное поле для поиска
        # defaults — значения для создания если товар не найден
        Product.objects.get_or_create(
            article=row['Артикул'],
            defaults={
                'name': row['Наименование товара'].strip(),
                'unit_type': row['Единица измерения'].strip(),
                'price': row['Цена'].strip(),
                'supplier': sup,
                'manufacturer': man,
                'category': cat,
                'discount': row['Действующая скидка'].strip(),
                'stock_quantity': row['Кол-во на складе'].strip(),
                'description': row['Описание товара'].strip(),
                'photo': row['Фото'].strip()
            }
        )
        
        # Выводим сообщение об успешной загрузке
        print(f"Успешно загружен {row['Артикул']}")
```

### 13.3 Скрипт импорта пользователей

**Путь:** `import_users.py`

```python
import django, csv, os

# Настройка Django
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'shoestore.settings')
django.setup()

# Импортируем модель пользователя
from products.models import User

# Открываем CSV с пользователями
with open('data/users.csv', 'r', encoding='utf-8-sig') as f:
    for row in csv.DictReader(f, delimiter=';'):
        # Создаём пользователя
        # create_user хэширует пароль автоматически!
        User.objects.create_user(
            username=row['Логин'].strip(),           # Имя пользователя
            password=row['Пароль'].strip(),          # Пароль (будет захэширован)
            first_name=row['ФИО'].split(' ')[0],     # Фамилия (первая часть ФИО)
            last_name=row['ФИО'].split(' ')[1] + ' ' + row['ФИО'].split(' ')[2],  # Имя и Отчество
            role=row['Роль сотрудника'].strip(),     # Роль
        )
        
        # Сообщение об успехе
        print(f"Успешно загружен {row['Логин']}")
```
